
# レベル 1 のエンタープライズ基本セキュリティ構成 

**適用対象**  

-   Windows 10

レベル 1 は、エンタープライズ デバイスの最小セキュリティ構成です。
Microsoft は、レベル 1 デバイスに対して次の構成を行うことを推奨しています。

## ハードウェア

レベル 1 をターゲットとするデバイスは、次のハードウェア機能をサポートする必要があります。

- [Trusted Platform Module (TPM) 2.0](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-tpm)
- [BitLocker ドライブ暗号化](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-bitlocker)
- [UEFI セキュア ブート](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-secure-boot)
- Windows Update を通じて配布されるドライバーとファームウェア

## ポリシー

レベル 1 のポリシーは、ユーザーまたはアプリケーションへの影響を最小限に抑えながら、適切なセキュリティ レベルを適用します。
Microsoft では、これらのセキュリティ構成と制御に[リング手法](https://docs.microsoft.com/windows/deployment/update/waas-deployment-rings-windows-10-updates)を使用することをお勧めします。セキュリティ制御の影響が限定的であるため、タイムラインは一般的に短くなる可能性があることに注意してください。

| 特徴 | ポリシー設定 | ポリシー値 | 説明 |

|---------|-------------------|-------------|
| OS セキュリティ アップデート | リリース後 7 日以内に Windows 品質更新プログラムを展開する | パッチのリリースからそのパッチのリバース エンジニアリングに基づく攻撃までの時間が短くなり続けているため、セキュリティ衛生の重要な側面は、セキュリティの脆弱性に対処する品質更新を迅速に検証および展開するエンジニアリング プロセスを持っていることです。 |
