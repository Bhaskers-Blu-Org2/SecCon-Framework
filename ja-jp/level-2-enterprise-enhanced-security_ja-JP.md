# レベル 2 のエンタープライズ拡張セキュリティ構成

**適用対象**  

-   Windows 10

レベル 2 は、ユーザーがより機密性の高い情報にアクセスするデバイスの標準として推奨されるセキュリティ構成です。これらのデバイスは、最近の企業では当然の標的になります。これらの推奨事項は、高度なセキュリティ レベルを対象としていますが、高度なスキルを持つ大規模なセキュリティ専門家を想定していません。したがって、ほとんどの企業組織からアクセスできるようにする必要があります。
レベル 2 の構成には、すべてのレベル 1 の構成が含まれ、以下のセキュリティ ポリシー、制御、および組織の動作が追加されます。

## ハードウェア

レベル 2 をターゲットとするデバイスは、レベル 1 のすべての機能をサポートし、次のハードウェア機能を追加する必要があります。

- [仮想化と HVCI が有効](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-vbs)
- [ドライバーとアプリ HVC I対応](https://docs.microsoft.com/windows-hardware/test/hlk/testref/driver-compatibility-with-device-guard)
- [Windows Hello](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello-biometric-requirements)
- [DMA I/O 保護](https://docs.microsoft.com/windows/security/information-protection/kernel-dma-protection-for-thunderbolt) 

## ポリシー

レベル 2 で適用されるポリシーには、レベル 1 で推奨されるすべてのポリシーが含まれ、以下のポリシーを追加して、
レベル 1 よりも多くの制御とより高度なセキュリティ構成を実装します。
ユーザーやアプリケーションへの影響がわずかに大きくなる可能性がありますが、
機密情報へのアクセス権を持つユーザーが直面するリスクに見合ったセキュリティ レベルを適用します。
Microsoft では、これらのセキュリティ構成と制御に[リング方法](https://docs.microsoft.com/windows/deployment/update/waas-deployment-rings-windows-10-updates)
を使用することをお勧めします。適度なタイムラインは、レベル 1 のプロセスよりも少し長くなることが予想されます。

## コントロール

レベル 2 で適用されるコントロールは、レベル 1 よりも多くのコントロールとより高度なセキュリティ構成を実装します。
\ユーザーやアプリケーションへの影響がわずかに大きくなる可能性がありますが、
機密情報へのアクセス権を持つユーザーが直面するリスクに見合ったセキュリティ レベルを適用します。
Microsoft は、監査モードのあるコントロールには監査/適用方法を使用し、
そうでないものには[リング方法](https://docs.microsoft.com/windows/deployment/update/waas-deployment-rings-windows-10-updates)
を使用することをお勧めします。また、適度なタイムラインは、レベル 1 のプロセスよりも少し長くなることが予想されます。

## 行動

レベル 2 で推奨される動作は、より高度なセキュリティ プロセスを実装します。
より洗練された組織を必要とする可能性がありますが、機密情報へのアクセス権を持つユーザーが直面するリスクに見合ったセキュリティ
レベルを適用します。

| 機能のセット | 特徴 | 説明 |
|------------|----------|--------------|
| ウィルス対策 | Microsoft からの取得にフェイルオーバーするように保護の更新を構成する | Windows Defender ウイルス対策の保護の更新のソースは、順序付けられたリストで提供できます。SCCM や WSUS などの内部配布を使用している場合は、リストの下にある Microsoft Update をフェイルオーバーとして構成します。 |
| OS セキュリティ更新 | 4日以内に Windows 品質更新プログラムを展開する | パッチのリリースとそのパッチのリバース エンジニアリングに基づく攻撃との間の時間は短くなりつつあるため、既知のセキュリティの脆弱性に対処する品質の更新を検証および展開する機能を提供するプロセスを設計することは、セキュリティ衛生の重要な側面です。|
| ヘルプデスク | 1 対 1 管理 | ヘルプデスク サポートのシンプルで一般的なモデルは、ヘルプデスク グループを各デバイスのローカル管理者グループの永続的なメンバーとして追加することです。デバイスが危険にさらされ、ヘルプデスクがそのデバイスに接続できる場合、これらの資格情報を使用して、他のすべてのデバイスに対する特権を取得できます。1 対 すべての管理者アクセスを提供せずにヘルプデスク サポートを提供する戦略を設計および実装する - これらのヘルプデスク資格情報の価値を制限する |
